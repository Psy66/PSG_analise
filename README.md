## Назначение и целевая аудитория

Приложение создано для **студентов и научных работников** (медицина, нейронауки, обработка сигналов), чтобы:

- загружать метаданные ПСГ‑исследований через REST API (только чтение);
- гибко фильтровать пациентов по клиническим критериям (возраст, AHI, ODI, SpO₂, коморбидности и др.);
- рассчитывать спектрально‑статистические признаки ЭЭГ (δ, θ, α, σ, β, γ, TBR, DAR, SE50, Sample Entropy) для тонических и фазических сегментов (подготовка данных);
- **проверять гипотезы** о влиянии апноэ на ЭЭГ с помощью **линейных смешанных моделей (LMM)** – единственный полностью реализованный метод анализа в текущей версии.

> **Основная научная задача** – поиск электрофизиологических маркеров, связанных с респираторными событиями, для потенциальной оценки кортикальной возбудимости и эффективности лечения СОАС.

## Основные возможности

### ✅ Доступно сейчас

#### 📡 Загрузка и фильтрация
- REST API (Bearer‑токен) – **только чтение**, никаких изменений на сервере.
- Гибкие критерии включения/исключения (возраст, AHI, ODI, SpO₂, качество записи, коморбидности: ХСН, эпилепсия, ИМТ>40, возраст>70, психотропные препараты и др.).

#### 📊 Описательная статистика
- Таблицы по группам тяжести ОАС (норма, лёгкая, умеренная, тяжёлая).
- Графики: распределение AHI, возраст, коморбидности.
- **Вкладка «Распределения»** – гистограмма + плотность, Q‑Q plot, boxplot по группам, автоматические рекомендации (параметрические / непараметрические тесты) на основе теста Шапиро‑Уилка и асимметрии.

#### 📈 Линейные смешанные модели (LMM) – тонический анализ
- **Тоническая модель**: влияние `has_apnea` на выбранные признаки (спектральные, статистические, нелинейные) + случайный эффект пациента, опционально фиксированный эффект стадии сна (N2/N3) и ковариаты (возраст, пол, ИМТ).
- FDR‑коррекция множественных сравнений (метод Бенджамини–Хохберга).
- Визуализация: лесной график (forest plot) для топ‑10 значимых признаков, вулкан‑график (volcano plot) для всех признаков.
- Диагностика остатков (Q‑Q plot, тест Шапиро‑Уилка, тест Бройша‑Пагана на гомоскедастичность).
- Блок‑бутстрап по пациентам (1000 итераций) для робастных доверительных интервалов.
- Генерация HTML‑отчёта с интерпретацией результатов и встроенными графиками.

### ⚠️ В разработке (будут добавлены в следующих версиях)

- **Фазический LMM анализ** (оценка реактивности бета, дельта‑энтропии в ответ на событие).
- **Анализ EDF‑файлов** (тонические и фазические признаки, position‑фильтр).
- **Event‑locked анализ** – усреднённые кривые γ‑активности вокруг респираторных событий.
- **PCA + логистическая регрессия** – снижение размерности и классификация эпох.
- **GAM (обобщённые аддитивные модели)** – нелинейные эффекты длительности события и времени.
- **DFA (детрендовый флуктуационный анализ) и когерентность ЭЭГ** – оценка фрактальных свойств и функциональной связности.
- **Классификация эпох и прогноз тяжести ОАС** с помощью машинного обучения (Random Forest, XGBoost).

## Планируемые возможности

Краткий анонс того, что появится в ближайших обновлениях:

- **Тонический и фазический анализ EDF** – загрузка локальных EDF‑файлов, расчёт признаков для эпох с апноэ и без, экспорт в CSV.
- **Event‑locked временные ряды** – визуализация динамики γ‑активности от –60 до +30 секунд относительно окончания события, расчёт пика, латентности, AUC.
- **PCA‑анализ** – визуализация разделимости классов «апноэ/без апноэ» в пространстве главных компонент.
- **GAM‑модели** (при наличии R и `mgcv`) – исследование нелинейных эффектов.
- **DFA и когерентность** – скейлинговый показатель α и кросс‑спектральная связность между отведениями.
- **Классификация эпох** – модели машинного обучения для детекции апноэ на 30‑секундных окнах.
- **Прогноз тяжести ОАС** – предсказание AHI (бинарная или порядковая классификация) на основе усреднённых ЭЭГ‑признаков и клинических данных.

Следите за обновлениями в репозитории.

## Установка и зависимости

**Требования:** Python 3.10+ (работает на Windows, Linux, macOS).

```bash
git clone https://github.com/Psy66/PSG_analise.git
cd PSG_analise
pip install -r requirements.txt
```

**Для GAM‑моделей** (опционально, потребуется в будущих версиях) дополнительно установите R (≥4.0) и пакет `mgcv`:
```r
install.packages("mgcv")
```

Затем установите `rpy2` (раскомментируйте в `requirements.txt`).

## Быстрый старт

Запуск GUI:
```bash
python main.py
```

- **Примечание по токену:** поле «Токен» не заполнено по умолчанию. Обратитесь к **Timur Petrenko (Psy66)**, чтобы получить действующий Bearer‑токен для доступа к исследованиям. Токен не является публичным и предоставляется только авторизованным исследователям.

### Типичный рабочий процесс (текущая версия)

1. **Вкладка «Загрузка и фильтры»** – подключитесь к API, примените фильтры.
2. **Вкладка «Статистика»** – посмотрите описательные характеристики выборки, оцените нормальность распределений.
3. **Вкладка «LMM анализ»** – выберите тип набора эпох (тонический, все эпохи, фильтр по положению), включите ковариаты, запустите анализ. Дождитесь результатов (таблица, forest plot, volcano plot). При необходимости выполните диагностику остатков для выбранного признака или бутстрап. Сформируйте HTML‑отчёт.

## Конфигурация (`core/config.py`)

Ключевые параметры (измените под свои нужды):

- `DEFAULT_TOKEN` – в репозитории оставлен пустым (или содержит пример, который не работает). **Реальный токен выдаётся администратором БД** (Timur Petrenko).
- `OUTPUT_DIR = "results"` – папка для выходных CSV и кэша.
- `TEST_MODE`, `MAX_PATIENTS` – тестовый режим.
- `MAX_PARALLEL_WORKERS` – для параллельной обработки (пока не используется).
- Частотные диапазоны: δ (0.5–4), θ (4–8), α (8–13), σ (12–15), β (15–30), γ (30–45) Гц.
- Параметры LMM, диагностики и бутстрапа настраиваются через GUI.

## Подробное описание вкладок

### 1. Загрузка и фильтры
- URL API, Bearer‑токен.
- Критерии: возраст, AHI, ODI, SpO₂, качество, коморбидности.
- Кнопка «Загрузить данные исследований» – получает метаданные из API.
- Кнопка «Применить фильтры» – сохраняет отфильтрованный DataFrame для использования в других вкладках.

### 2. Статистика
- Расчёт описательных статистик (n, возраст, AHI, SpO₂, TST, эффективность, коморбидности) по группам тяжести ОАС.
- Вкладка «Распределения» – анализ нормальности с графиками и рекомендациями.

### 3. LMM анализ (основной инструмент)
- **Выбор типа набора эпох:** тонический (data_type=1), все эпохи (2) или фильтр по положению (3). Данные загружаются из API на основе отфильтрованных исследований.
- **Настройки модели:** включение стадии сна (N2/N3) как фиксированного эффекта, включение ковариат (возраст, пол, ИМТ), порог FDR.
- **Запуск LMM** – перебираются все комбинации канал × признак (статистические, спектральные, TBR, DAR, SE50, SampEn, а также когерентность). Для каждой модели строится mixed‑model с фиксированным эффектом `has_apnea` и случайным перехватом `(1|patient_id)`.
- **Результаты:** таблица со столбцами channel, feature, beta, p_value, q_value, CI, n_obs. Лесной график (топ‑10 значимых) и вулкан‑график.
- **Проверка нормальности:** для выбранного канала и признака – Q‑Q plot, тест Шапиро‑Уилка.
- **Диагностика модели:** для выбранного признака – график остатков vs предсказанные, Q‑Q plot остатков, гистограмма остатков, тесты Шапиро‑Уилка и Бройша‑Пагана.
- **Бутстрап:** блок‑бутстрап по пациентам (1000 итераций) для робастных доверительных интервалов.
- **Отчёт:** генерация HTML‑файла с интерпретацией, таблицами, графиками и рекомендациями.

> **Важно:** В текущей версии **не реализованы** вкладки PCA, GAM, Event‑locked, DFA/когерентность. Их интерфейс закомментирован в коде и будет активирован после завершения разработки.
