# APNEA AI – анализ ЭЭГ-биомаркеров обструктивного апноэ сна

**APNEA AI** – научно-исследовательский инструмент для выявления нейрофизиологических ЭЭГ-биомаркеров обструктивного апноэ сна (ОАС) с использованием методов статистического анализа и машинного обучения. Проект реализует методы, описанные в **Главе 2** диссертационной работы, с фокусом на **раздел 2.5 (Статистический анализ)**.

## 🎯 Научная цель

Основная цель – выявить и количественно оценить изменения биоэлектрической активности мозга (ЭЭГ) при обструктивном апноэ сна, проверить следующие гипотезы:

| Гипотеза | Описание |
|----------|----------|
| **H1** (тоническая спектральная) | Относительная мощность дельта-ритма положительно коррелирует с тяжестью ОАС (AHI, ODI, T90), особенно в лобных отведениях. |
| **H2** (тоническая спектральная) | Сигма-мощность и отношение тета/бета (TBR) достоверно снижаются при нарастании тяжести ОАС. |
| **H3** (фазическая) | Гамма-мощность и индекс реактивности бета в пост-событийном окне значимо выше в эпохах с апноэ, коррелируют с длительностью события и AHI. |
| **H4** (нелинейная) | Sample Entropy (SampEn) снижается в период апноэ и резко возрастает после окончания события. |
| **H5** (классификация эпох) | Комплекс признаков ЭЭГ позволяет классифицировать 30‑с эпохи N2/N3 на «апноэ» vs «без апноэ» с AUC ≥ 0,85. |
| **H6** (прогноз тяжести) | Агрегированные ЭЭГ-признаки + клинические данные предсказывают тяжесть ОАС (AHI<15 vs ≥15) с точностью ≥80%. |

Проверка гипотез выполняется при уровне значимости **q < 0,05** после FDR-коррекции множественных сравнений (метод Бенджамини – Хохберга).

## ✨ Основные возможности

- **Загрузка и фильтрация данных**  
  – Получение ПСГ-исследований, пациентов и клинических данных через REST API.  
  – Применение критериев включения/исключения (возраст, длительность записи, качество, AHI, ODI, SpO₂, коморбидности и др.).

- **Статистический анализ (п. 2.5)**  
  - **Линейные смешанные модели (LMM)** для сравнения эпох «с апноэ» и «без апноэ» с учётом ковариат (возраст, пол, ИМТ, стадия сна).  
  - FDR-коррекция, диагностика остатков (нормальность, гомоскедастичность) и блок-бутстрап (1000 итераций).  
  - **PCA + LMM для главных компонент** и ROC‑анализ.  
  - **Иерархическая кластеризация пациентов** (метод Уорда) на основе средних PC1/PC2, оценка устойчивости бутстрапом.  
  - **Генерализованные аддитивные модели (GAM)** для нелинейных зависимостей γ-мощности от длительности события и времени (интеграция с R/mgcv).  
  - **DFA и когерентность** – анализ долговременной корреляции и функциональной связности ЭЭГ.  
  - **Event‑locked анализ** – динамика нормализованной γ-активности относительно окончания события (с построением средних кривых по группам тяжести).  

- **Машинное обучение** (реализовано во вкладках `PCA`, `GAM`, `LMM` как предварительные этапы; полноценное ML – в разработке/вынесено в отдельные скрипты).

- **Визуализация и отчёты**  
  – Forest plot, volcano plot, контурные графики GAM, матрицы когерентности, дендрограммы, Q‑Q plots, гистограммы.  
  – Автоматическая генерация HTML-отчётов со встроенными графиками и интерпретацией гипотез.

## 💾 База данных

Проект использует реляционную БД **MariaDB** со следующей ключевой схемой:

| Таблица | Содержание |
|---------|-------------|
| `epoch_features` | 30‑секундные эпохи N2/N3: статистические, спектральные, нелинейные признаки, DFA, когерентность (180+ колонок). |
| `phasic_events` | Фазические признаки (γ-мощность, β‑реактивность, ΔSampEn) для каждого респираторного события. |
| `event_time_series` | Временные ряды нормализованной γ‑мощности (интервал -60…+30 с относительно offset). |
| `sleep_statistics` | Агрегированные показатели сна: AHI, ODI, SpO₂, TST, эффективность сна. |
| `patient_clinical_data` | Клинические данные: коморбидности, антропометрия, лекарства, лаборатория. |

Доступ к БД осуществляется через **REST API** (адрес и токен задаются в GUI).

## 🔧 Установка и запуск

### Требования
- **Python 3.14+**
- **MariaDB/MySQL** сервер с развёрнутой базой `PSG` (структура в файле `PSG.sql`).
- **R** (≥4.3) с пакетом `mgcv` – только для работы вкладки GAM (опционально).

### Установка зависимостей
```bash
git clone https://github.com/Psy66/PSG_analise.git
cd PSG_analise
pip install -r requirements.txt
```

Основные библиотеки: `numpy`, `pandas`, `scipy`, `statsmodels`, `scikit-learn`, `matplotlib`, `seaborn`, `mne`, `nolds`, `antropy`, `requests`, `rpy2` (опционально), `tkinter` (встроен).

### Настройка
1. Убедитесь, что сервер БД запущен и API-сервер (FastAPI/Flask) принимает запросы по адресу `https://med.smed96.synology.me/psgapp/api/`.
2. Получите API-токен у администратора системы.
3. При первом запуске введите адрес API и токен во вкладке **«Загрузка и фильтры»**.
4. При необходимости измените пути и параметры в `core/config.py` (директории для кэша, диапазоны фильтрации и пр.).

### Запуск
```bash
python main.py
```

## 📖 Использование (краткое руководство)

1. **Загрузка и фильтры**  
   – Введите `Адрес API` и `Токен`, нажмите **«Загрузить данные исследований»**.  
   – Установите критерии включения/исключения (возраст, AHI, качество записи, коморбидности и т.д.).  
   – Нажмите **«Применить фильтры»**. Отфильтрованный DataFrame появится в главном окне.

2. **Статистика**  
   – Рассчитайте описательные статистики по группам тяжести ОАС.  
   – Постройте гистограммы, Q-Q plots для проверки нормальности признаков.

3. **LMM анализ**  
   – Выберите режим: «Сравнение эпох» (апноэ vs без) или «Тонические признаки vs AHI».  
   – Запустите LMM для всех каналов и признаков. Результаты (β, p-value, q-value, доверительные интервалы) отобразятся в таблице.  
   – Для выбранного признака выполните диагностику остатков, блок-бутстрап (1000 итераций).  
   – Сформируйте HTML-отчёт с интерпретацией гипотез.

4. **PCA и кластеризация**  
   – Задайте типы эпох для PCA (обычно тонические) и для LMM (смешанные, содержащие апноэ).  
   – Запустите PCA → отобразятся loadings, объяснённая дисперсия, проекции.  
   – Выберите компоненты для кластеризации, настройте метод/метрику и нажмите «Перекластеризовать». Результаты (дендрограмма, сравнение кластеров) появятся на вкладке.

5. **Event‑locked анализ**  
   – Выберите канал (C3/C4) и временной интервал.  
   – Нажмите «Запустить анализ». Построятся средние кривые γ‑мощности для групп тяжести ОАС с 95% ДИ.  
   – Рассчитаются метрики (пик, латентность, AUC) и выполнится бутстрап-сравнение групп.

6. **DFA и когерентность**  
   – Режим: LMM для DFA-экспонента или для когерентности (пары отведений × частотные диапазоны).  
   – Построение карты связей (brain map) и тепловой матрицы когерентности.  
   – ANOVA для DFA по группам тяжести ОАС.

7. **GAM анализ**  
   – Требуется R и `mgcv`. Оценивает нелинейную зависимость γ‑мощности от времени и длительности события.  
   – Результаты: summary модели, таблица гладких членов, контурный график, частичные зависимости.

## 📊 Примеры выходных данных

- **Таблица LMM**: `channel`, `feature`, `beta`, `p_value`, `q_value`, `ci_low`, `ci_high`, `significant`.
- **Forest plot** – топ-10 значимых признаков с 95% CI.
- **Volcano plot** – зависимость `-log10(p)` от `beta`, цветом выделены значимые после FDR.
- **Матрица когерентности** – средние значения для выбранной группы и диапазона.
- **Кривые event-locked** – динамика γ-активности (с доверительными интервалами) для групп тяжести.
- **Отчёт HTML** – включает все графики, таблицы, проверку гипотез и статистические тесты.
